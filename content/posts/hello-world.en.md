Hi, I'm Seunghan, a developer. I'm starting the seunghan.xyz tech blog.

This blog is a space to record problems I've encountered while developing and how I solved them. Rather than polished knowledge, I want to honestly document real debugging experiences and what I learned from them.

---

## Why I Started This Blog

When developing, it's common to spend hours solving a single error. Googling often doesn't yield the exact answer, and Stack Overflow answers frequently don't apply because of version differences.

I felt it was a waste to just forget about problems I'd solved. I'm starting this record for when I encounter the same problem again in the future, and hoping it might help other developers in similar situations.

---

## Main Tech Stack

Here are the technologies I mainly work with:

**Mobile**
- Flutter / Dart — iOS, Android cross-platform apps
- Swift / SwiftUI — iOS native
- Hotwire Native — Rails-based hybrid apps

**Backend**
- Ruby on Rails 8 — API servers and full-stack web apps
- PostgreSQL — primary database
- Render, AWS — cloud deployment

**Frontend**
- Svelte 5 + Inertia.js — SPA integrated with Rails
- React / Next.js — standalone web services
- Tailwind CSS — styling

---

## Topics I'll Cover

**Debugging records** will probably be the most common. I plan to mainly cover things that are hard to solve with official docs alone, like Flutter TestFlight deployment issues, Rails OAuth integration errors, and iOS build configuration problems.

Additionally:

- Side project retrospectives — from idea to App Store launch
- Tech choice rationale — why I chose this stack
- Development productivity — tools and methods that improved my workflow
